TypeScript에서의 Class와 Interface

TypeScript에서의 Class와 Interface 차이, 활용

2025.03.01

10mins

TypeScript
Class
Interface

Class

Class란?


객체를 생성하는 템플릿 상태(프로퍼티), 동작(메서드) 포함



특징

  • constructor를 사용해 객체 초기화 가능
  • private, protected, public 등의 접근 제어자를 사용 가능
  • 메서드 포함 가능
  • 상속 가능

예시

Interface

Interface란?


객체의 구조(타입) 만을 정의, 구현을 강제하는 역할



특징

  • 실제 구현을 포함하지 않음
  • 클래스나 객체가 특정 구조를 따르도록 강제
  • implements를 사용하여 클래스가 인터페이스를 따르도록 설정 가능
  • 컴파일 시 사라짐 -> 런타임 성능에 영향을 주지 않음
  • 유지보수성과 확장성에 용이

정리

비교 항목클래스 (Class)인터페이스 (Interface)
목적객체 생성 및 구현 정의객체의 구조(형식)만 정의
구현 가능 여부구현 가능 (메서드, 생성자 등)구현 불가 (구조 정의만 가능)
인스턴스✅ new를 통해 인스턴스를 생성 가능❌ 인스턴스 생성 불가
메서드 포함 여부✅ 가능 (메서드 구현 가능)✅ 선언 가능 (구현 X)
접근 제어자✅ private, protected, public 사용 가능❌ 접근 제어자 사용 불가
상속 (Inheritance)✅ extends 사용✅ extends, implements 사용
다중 상속❌ 다중 상속 불가✅ 다중 상속 가능
컴파일 후 결과✅ JavaScript의 class로 변환됨❌ 사라짐 (타입 체크용)
