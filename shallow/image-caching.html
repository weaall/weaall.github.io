이미지 캐싱이란?


이미지 캐싱은 웹 페이지의 속도와 트래픽 효율을 높이기 위해 이미지를 여러 계층(브라우저 캐시, CDN, S3 등)에 임시 저장하는 기술입니다. 사용자는 가까운 CDN이나 브라우저 캐시, Service Worker를 통해 이미지를 빠르게 받아볼 수 있습니다.



이미지 캐싱의 장점


  • 속도 향상: 이미지를 로컬 캐시에서 불러오기 때문에 페이지 로딩 속도가 빨라집니다.
  • 서버 부하 감소: 동일한 이미지를 여러 번 요청하지 않으므로 서버의 부하가 줄어듭니다.
  • 네트워크 대역폭 절약: 이미지를 재다운로드하지 않으므로 네트워크 대역폭을 절약할 수 있습니다.


이미지 캐싱 구조


  1. 첫 요청: 브라우저가 이미지를 요청하면, CDN에 캐시가 없을 경우 S3에서 이미지를 받아옵니다.
  2. CDN 캐싱: CDN은 이미지를 엣지 서버에 저장해, 다음 요청부터 빠르게 제공합니다.
  3. Service Worker 캐싱: 브라우저의 Service Worker가 이미지를 로컬에 저장, 재방문시 캐시 이미지 제공
  4. 후속 요청: 우선 Service Worker, 그다음 CDN, 마지막으로 S3에서 이미지를 받아 최적의 속도를 냅니다.

이미지 캐싱 전략


대규모 웹 서비스에서는 이미지 캐싱 효율을 극대화하기 위해 Object Storage(S3), CDN(CloudFront), Service Worker를 조합한 3단계 캐싱 전략을 사용


이미지 캐싱 구조

0단계: 이미지 구성

  • Thumbnail, Medium, Large 등 다양한 크기의 이미지를 준비
  • 각 크기에 맞는 최적화된 이미지 포맷 사용
    (WebP : GIF 대비 최대 64%, PNG-JPEG 대비 평균 25%, 최대 36% 용량 절감)
  • 각 이미지에 대해 적절한 메타데이터와 캐시 정책 설정

1단계: Object Storage (S3)

  • 이미지를 안전하게 저장하는 기본 저장소 역할
  • 버전 관리, 접근 제어, 내구성 보장
  • 직접적으로 사용자에게 이미지를 제공하지 않고, CDN이 S3에서 이미지를 가져감

2단계: CDN(CloudFront)

  • S3에 저장된 이미지를 전 세계 엣지 서버에 분산 캐싱
  • 사용자는 가장 가까운 CDN 엣지 서버에서 이미지를 빠르게 다운로드 (수십 밀리초에서 수 초까지 로딩 시간 단축)
  • HTTP 캐시 헤더(Cache-Control, Expires)를 통해 브라우저와 CDN의 캐싱 정책을 세밀하게 제어

3단계: Service Worker

  • 사용자의 브라우저에서 동작하는 JS 기반 프록시
  • 네트워크 요청을 가로채어, 이미지 등 정적 리소스를 로컬 캐시에 저장 (로딩시간을 거의 0ms로 단축, 이미지 전송량 0)
  • 오프라인 환경에서도 이미지 제공
  • 캐시 만료, 업데이트 등 세밀한 제어 가능
  • 추가로 서비스워커는 클라이언트 이미지들도 캐싱하므로 SVG, Logo 호출에 대한 부담감소

캐싱 상황


한 유저의 이미지 10만번 호출 (캐싱 적중률 100%)

  1. S3
  • S3 : 100,000
  • 부하 증가로 인한 네트워크 지연
  1. S3 + CDN
  • S3 : 1, CDN: 100,000
  • 비용 감소 및 네트워크 지연 최소화 (사용자 경험 상승)
  1. S3 + CDN + Service Worker
  • S3 : 1, CDN: 1, Service Worker: 99,999
  • 로딩 시간 0ms, 이미지 전송량 0

한 유저의 이미지 10만번 호출 (캐싱 적중률 80%)

  1. S3
  • S3 : 100,000
  • 부하 증가로 인한 네트워크 지연
  1. S3 + CDN
  • S3 : 20,000, CDN: 100,000
  • 비용 감소 및 네트워크 지연 최소화 (사용자 경험 상승)
  1. S3 + CDN + Service Worker
  • S3 : 20,000, CDN: 20,000, Service Worker: 80,000
  • 로딩 시간 0ms, 이미지 전송량 0

1000명 유저들의 100개의 이미지 100번씩 호출

  1. S3
  • S3 : 10,000,000
  • 부하 증가로 인한 네트워크 지연
  1. S3 + CDN
  • S3 : 100, CDN: 10,000,000
  • 비용 감소 및 네트워크 지연 최소화 (사용자 경험 상승)
  1. S3 + CDN + Service Worker
  • S3 : 100, CDN: 100,000, Service Worker: 9,900,000
  • 로딩 시간 0ms, 이미지 전송량 0

결론


  • 이미지 캐싱전략은 그 자체로 사용자 경험 개선과 비용 절감에 엄청난 효과를 가져오므로, 대규모 웹 서비스에서는 필수적으로 적용해야 한다고 생각합니다.
