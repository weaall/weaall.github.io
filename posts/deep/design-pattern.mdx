## 디자인 패턴 정리

### 생성 패턴 (Creational Patterns)
객체를 효율적으로 만들고 관리하는 방법을 제공

| 패턴 | 설명 |
|------|------|
| **Singleton** | 단 하나의 인스턴스만 생성하고 공유하는 패턴 |
| **Factory Method** | 객체 생성을 위한 인터페이스를 제공하고, 하위 클래스가 인스턴스를 생성하도록 위임 |
| **Abstract Factory** | 관련된 객체 그룹을 생성할 수 있도록 하는 패턴 |
| **Builder** | 객체 생성을 단계별로 진행하여 복잡한 객체를 쉽게 만들 수 있도록 지원 |
| **Prototype** | 기존 객체를 복제하여 새로운 객체를 생성 |

### 구조 패턴 (Structural Patterns)
객체 간의 관계를 정의하고 효율적으로 연결하는 패턴

| 패턴 | 설명 |
|------|------|
| **Adapter** | 서로 다른 인터페이스를 가진 클래스들을 연결할 때 사용 |
| **Bridge** | 구현과 추상화를 분리하여 독립적으로 확장 가능하도록 함 |
| **Composite** | 트리 구조를 만들어 개별 객체와 그룹을 동일하게 다룰 수 있도록 함 |
| **Decorator** | 기존 객체에 기능을 추가할 때 사용 |
| **Facade** | 복잡한 서브 시스템을 단순한 인터페이스로 감싸는 패턴 |
| **Flyweight** | 메모리 사용을 줄이기 위해 공유 가능한 객체를 활용 |
| **Proxy** | 객체 접근을 제어하기 위해 대리 객체를 사용 |

### 행동 패턴 (Behavioral Patterns)
객체 간의 상호작용을 정의하고, 책임을 효과적으로 분배하는 패턴

| 패턴 | 설명 |
|------|------|
| **Observer** | 한 객체의 상태 변화가 여러 객체에게 자동으로 전달됨 (이벤트 리스너 같은 구조) |
| **Strategy** | 실행 중에 알고리즘을 변경할 수 있도록 인터페이스를 제공 |
| **Command** | 요청을 객체로 캡슐화하여 실행 취소 및 재사용이 가능하도록 함 |
| **State** | 객체의 상태를 캡슐화하여 상태 변화에 따른 동작을 변경 |
| **Template Method** | 알고리즘의 구조를 정의하고, 세부 구현은 서브클래스에서 결정 |
| **Mediator** | 객체 간의 복잡한 의존 관계를 중앙에서 관리 |
| **Chain of Responsibility** | 요청을 처리할 수 있는 객체를 체인 형태로 연결하여 유연한 요청 처리를 가능하게 함 |
| **Visitor** | 객체의 구조를 변경하지 않고 새로운 동작을 추가 |
